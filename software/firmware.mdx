## Políticas de actualización

- Programa ventanas de mantenimiento trimestrales para aplicar nuevas versiones.
- Valida siempre la firma digital SHA-512 provista por ZeroKey.
- Habilita el modo de bloqueo remoto para impedir el uso del dispositivo durante la actualización.
- Conserva la versión previa durante 30 días para procedimientos de rollback.

## Flujo automatizado

<NumberedList>
  <li>
    Descarga el paquete `.zkfw` desde el portal de clientes y súbelo al repositorio seguro.
  </li>
  <li>
    Ejecuta `zerokey cli firmware validate paquete.zkfw` para comprobar la integridad.
  </li>
  <li>
    Distribuye el firmware a los agentes usando `zerokey cli firmware push --group="Piloto"`.
  </li>
  <li>
    Monitoriza el progreso con `zerokey cli firmware status --watch` hasta alcanzar el 100%.
  </li>
</NumberedList>

## Actualización manual (modo rescate)

1. Conecta la llave y mantén presionado el botón capacitivo durante 5 segundos.
2. Ejecuta `zerokey cli firmware flash --device=<serie> --file=paquete.zkfw`.
3. Espera a que el LED amarillo deje de pulsar.
4. Desconecta y vuelve a insertar la llave para finalizar el proceso.

<Tip>
  Si el proceso se interrumpe, el dispositivo vuelve automáticamente a la versión anterior. Repite el comando una vez restablecida la conectividad.
</Tip>

## Matriz de compatibilidad

| Firmware | Sistemas operativos compatibles | Características clave |
| --- | --- | --- |
| 2.3.x | Windows 10/11, macOS 13+, Ubuntu 22.04 | Mejora de FIDO2, soporte OTP offline |
| 2.2.x | Windows 10/11, macOS 12+, Ubuntu 20.04 | Registro biométrico, auditoría extendida |
| 2.1.x | Windows 10, macOS 11, Debian 11 | Liberación inicial con soporte básico OTP |

## Alertas y monitoreo

Utiliza la integración con tu SIEM para recibir alertas cuando:

- Un dispositivo no se actualiza después de 3 intentos.
- Se detecta un firmware no autorizado.
- Se supera el límite de tiempo establecido para la ventana de mantenimiento.
- Un usuario pospone la actualización más de dos veces consecutivas.
