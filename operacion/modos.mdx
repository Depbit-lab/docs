## Descripción general

ZeroKeyUSB admite múltiples perfiles para adaptarse a las políticas de acceso de tu organización. Puedes alternar los modos de forma remota o local mediante la consola.

## Perfiles disponibles

| Modo | Uso recomendado | Detalles |
| --- | --- | --- |
| Autenticación FIDO2 | Acceso a aplicaciones web y VPN | Requiere PIN o biometría, compatible con navegadores modernos. |
| OTP temporal | Sistemas heredados y servicios sin soporte WebAuthn | Genera códigos de 6 u 8 dígitos sincronizados por tiempo. |
| Firmas digitales | Aprobaciones de documentos y transacciones | Utiliza certificados X.509 almacenados en la llave. |
| Desbloqueo físico | Control de acceso a puertas y racks | Emulación HID para lectores compatibles con HID iCLASS. |

## Configuración de perfiles

<Steps>
  <Step title="Selecciona el dispositivo">
    Desde ZeroKey Console, abre **Dispositivos → Inventario** y elige la llave objetivo.
  </Step>
  <Step title="Activa el modo">
    Dentro de **Perfiles**, habilita el modo deseado y ajusta parámetros como duración del OTP, longitud del PIN o certificados asignados.
  </Step>
  <Step title="Sincroniza políticas">
    Presiona **Guardar y desplegar** para que el cambio se aplique en el próximo contacto del agente (aprox. 60 segundos).
  </Step>
</Steps>

## Automatización mediante CLI

```bash
# Consultar modo activo
zerokey cli device profile --device=ZK-2045

# Cambiar a modo OTP con expiración de 30s
zerokey cli device profile set --device=ZK-2045 --mode=otp --ttl=30

# Habilitar firma digital y adjuntar certificado
zerokey cli device profile set --device=ZK-2045 --mode=signature --cert=/seguro/certificado.pfx
```

## Recomendaciones

- Mantén deshabilitados los modos que no utilices para reducir la superficie de ataque.
- Define grupos de dispositivos por área (Finance, IT, Ventas) para aplicar perfiles consistentes.
- Documenta los cambios en tu sistema ITSM para mantener trazabilidad.
- Valida cada perfil en un entorno de prueba antes de desplegarlo en producción.
