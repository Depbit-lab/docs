## Controles recomendados

- **PIN complejo**: mínimo de 6 dígitos con historial de reutilización.
- **Bloqueo progresivo**: incrementa el tiempo de espera tras cada intento fallido.
- **Requerir presencia del usuario**: activa la detección táctil capacitiva en todos los flujos.
- **Contexto adaptativo**: condiciona el acceso según ubicación, dispositivo o dirección IP.

## Políticas en ZeroKey Console

<NumberedList>
  <li>
    Ingresa a **Configuración → Políticas** y crea un nuevo perfil.
  </li>
  <li>
    Define los umbrales de seguridad (PIN, biometría, presencia, límites de sesión).
  </li>
  <li>
    Asocia el perfil a grupos de usuarios o unidades organizativas.
  </li>
  <li>
    Programa auditorías automáticas para recibir reportes semanales.
  </li>
</NumberedList>

## Integración con SIEM

Para enviar eventos a tu plataforma de monitoreo:

```bash
zerokey cli audit forward --destination=syslog://10.10.4.20:514 --facility=auth
```

Los eventos incluyen registros de autenticación, cambios de políticas y alertas de hardware.

## Requisitos de cumplimiento

| Marco | Controles aplicables | Evidencia disponible |
| --- | --- | --- |
| ISO/IEC 27001 | A.9 Control de acceso, A.12 Operación segura | Informes de auditoría y registros de autenticación |
| PCI-DSS 4.0 | 8.4 MFA en acceso administrativo | Registros de tokens FIDO2 y políticas de PIN |
| GDPR | Art. 32 Seguridad del tratamiento | Cifrado de datos y controles de acceso basados en roles |

## Respuesta a incidentes

1. Revoca el dispositivo comprometido desde **Dispositivos → Acciones → Bloquear**.
2. Marca el incidente en el sistema ITSM y adjunta logs relevantes.
3. Despliega un reemplazo siguiendo el flujo de inicio rápido.
4. Ejecuta un análisis posterior para ajustar políticas preventivas.
