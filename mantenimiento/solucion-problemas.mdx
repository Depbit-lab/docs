## Errores frecuentes

| Código | Descripción | Resolución |
| --- | --- | --- |
| ZK-101 | Dispositivo no reconocido por el sistema | Verifica puertos USB, reinstala controladores y prueba en otro equipo. |
| ZK-204 | PIN incorrecto superó el límite | Restablece el dispositivo desde la consola con respaldo de códigos de recuperación. |
| ZK-309 | Firmware desactualizado | Ejecuta `zerokey cli firmware push --device=<serie>` para aplicar la última versión. |
| ZK-412 | Integración con IdP fallida | Revisa credenciales del conector y sincroniza nuevamente los metadatos. |

## Procedimiento de recuperación

1. Bloquea temporalmente el dispositivo afectado.
2. Notifica al usuario e inicia el flujo de verificación de identidad.
3. Si procede, realiza un restablecimiento seguro:

```bash
zerokey cli device reset --device=ZK-2045 --preserve-audit
```

4. Reasigna el dispositivo o programa un reemplazo.

## Escalamientos al soporte de ZeroKey

Cuando abras un ticket, incluye:

- Número de serie del dispositivo.
- Logs exportados con `zerokey cli device logs --device=<serie> --full`.
- Fecha y hora aproximada del incidente (UTC).
- Impacto en usuarios y aplicaciones.

## Preguntas comunes

<AccordionGroup>
  <Accordion title="¿Cómo restablezco el PIN si el usuario lo olvidó?">
    Utiliza `zerokey cli device pin reset --device=<serie>` y entrega un PIN temporal que debe cambiarse en el siguiente uso.
  </Accordion>
  <Accordion title="¿Puedo usar ZeroKeyUSB en modo offline?">
    Sí, el modo OTP temporal permite autenticar sin conexión durante 7 días con sincronización posterior.
  </Accordion>
  <Accordion title="¿Qué hago si el LED permanece rojo?">
    Indica que se alcanzó el límite de intentos fallidos. Requiere restablecimiento remoto desde la consola.
  </Accordion>
</AccordionGroup>
